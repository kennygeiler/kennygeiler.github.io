Week 1 Recap

All about version control & Git/Github

Friday the 16th, 2015

Version control is an extremely useful tool for developers. Using version control allows developers to venture and make changes to code without worrying about altering previous code. In addition, version control saves all edits so if mistakes are made, developers can go back to previous versions of the code. Git, which is a version control system, helps keep track of these changes. Whenever a change is made to code or a file, it is logged in git and the user can check all of the previous changes through git. Paired with git is GitHub. While git stores all of your versions locally on your computer, GitHub stores them online for you to share with your peers. This is essential for teams that work on the same code and need to view eachother's work seamlessly and easily.

Arrays & Hashes

All about arrays and hashes

The comparing and contrasting of the different aspects of arrays and hashes

Friday, January 30th, 2015

In Ruby, arrays and hashes are very useful tools for working with lists of stored information. Arrays and hashes can hold different types of objects ranging from strings to integers to floats. An array is an ordered list, whose position can be asseced through integers. A hash is more of a collection of key/value pairs. Hashes point objects to other objects giving them meaning while arrays are an ordered list of objects. Therefore, it is easier to access elements of an array, but the use of hashes results in more flexibility.

The Ruby method that I will be researching today is #cycle. After reading the Ruby Docs on this method, it seems that is it a powerful method that could be used in place of a while loop. Sunday, Feburary 8th, 2015

One of the biggest problems facing the tech world today has nothing to do with technology itself. How is that possible one may ask? Well, it turns out, a major problem with the tech world is its culture. It is no secret that the tech world is domianted by males. There are countless tales of women being outnumbed and uncomfortable in the workplace. Women report feeling discrimination based on their age, race, sexuality, and the prospects of motherhood. In a fortune.com article surverying 716 women in the tech industry who have left the tech industry who have left mainly because they have no desire to work in the industry. Is it because its too hard? No. The major reason is the workplace environment was so bad that 465 of the women survery ended up not working at all, 251 are working in a non-tech job, and 45 of them are now running their own companies. One may think there are a lack of women in tech, which can cause them to be outnumbered. Incorrect. Top computer science colleges like Stanford accept apx. 50% males and females. The only way to fix this problem is to fix the entire culture of the tech industry workplace, and destory the out-dated views of male dominance in the workplace.

Technical Blog Post #5

All about classes

Classes, instance variables, methods, and more!

Sunday, Febuary 15th

Classes are the best. When you need to create something that can serve a purpose for different inputs at different times without re-writing methods every time, a class is the ideal solution. When you create a class you are creating a block of code that holds methods and variabls that can be called on different inputs over and over. For example, a class I recently created tested the valididity of a credit card number. Therefore, any time I am given a number, I can create a instance of that class and call the methods I wrote to test that given number. The best part about classes is the ability to carry variables through different methods and modify them. These are called instance variables. They are defined once at the top of a class and you can call them throughout in any method and modify them along the way. You make make your life even eaier with attributes that allow varaibles to be written and called througout a class without having to create a method for them.

Technical Blog Post #6

All about variable scope

The different types of variables and how/why that affects your code

Sunday, Febuary 22nd

Variable scope is an important concept in ruby and programming as a whole. Classes, methods, and more depend on the inputs and modifications of variables. These variables have to be properly created and scoped so that they can be accessed at the right place and not accidently altered in the wrong ones. The variable scope catagories are: global, instance, local, constant, and class variable. A local variable is one that is created in a method or a loop and can only be accessed within that method / loop. On the other hand, a global variable is accessible from anywhere in the program regardless of where they are declared. These variables have a $ sign in front of them. A class variable is one that is shared for all objects instantiated within a class. Instance variables can be changed within a class but do not work outside the class. A constant variable is one who's value should not be changed and says "constant" throughout the code.
